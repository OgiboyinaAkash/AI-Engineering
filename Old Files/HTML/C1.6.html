Capstone — Domain AI Assistant

Capstone Overview

This is it. Everything you've learned across Track 1 — how LLMs work under the hood (C1.1), how to craft reliable prompts (C1.2), how to integrate APIs with streaming and error handling (C1.3), how to give LLMs tools to act in the world (C1.4), and how to ground responses in real documents using RAG (C1.5) — all comes together here.

You're going to build a domain-specific Q&A assistant that can answer questions accurately over a small document set. Not a toy demo. A working system with real prompts, real retrieval, real evaluation, and a real demo.

But wait — haven't we already built something like this in C1.5?

Yes, but the C1.5 lab was a guided walkthrough with mock data. This capstone is different. You choose the domain. You curate the documents. You design the prompts. You measure the quality. You improve what's broken. And you present your work to a reviewer like you would to a tech lead at a company.

Think of C1.5 as learning to ride a bicycle in a parking lot. The capstone is your first real ride through the city.

Choose Your Domain

Pick one of the three domains below. Each comes with a pre-curated document set and evaluation queries so you can focus on building, not on data collection.

Domain Document Set Example User Queries
Legal Advisor 20 clauses from a standard employment agreement: termination, non-compete, leave policy, IP ownership, confidentiality, etc. "Can I work for a competitor after leaving?" / "What happens to my code if I quit?" / "How many sick leaves do I get?"
Health FAQ Bot 25 entries from a clinic's patient guide: appointment booking, insurance coverage, vaccination schedules, emergency protocols, medication refills. "How do I book an appointment with a dermatologist?" / "Is dental covered under the basic plan?" / "What vaccines does a 2-year-old need?"
University Course Guide 30 entries covering course descriptions, prerequisites, credit requirements, exam policies, grading rubrics, and faculty office hours. "What are the prerequisites for Machine Learning?" / "How many credits do I need to graduate?" / "What's the late submission penalty?"

Can I pick my own domain? You can, but only if your document set has at least 20 entries and you can write 15 evaluation queries for it. Scoping a domain is harder than it looks — the pre-curated options save you hours.

Starter Code Skeleton

import os
import json
import chromadb
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()

class DomainAssistant:
    def __init__(self, domain_name, documents):
        self.llm = OpenAI()
        self.chroma = chromadb.Client()
        self.collection = self.chroma.create_collection(name=domain_name)
        self.total_cost = 0.0

        # Load documents
        self.collection.add(
            documents=documents,
            ids=[f"doc_{i}" for i in range(len(documents))]
        )
        print(f"[{domain_name}] Loaded {len(documents)} documents.")

    def retrieve(self, question, top_k=3):
        results = self.collection.query(query_texts=[question], n_results=top_k)
        return results["documents"][0], results["ids"][0]

    def build_prompt(self, context_chunks, question):
        context = "\n---\n".join(context_chunks)
        return f"""Answer the question based ONLY on the context below.
If the answer is not in the context, say "I don't have information about that."

Context:
{context}

Question: {question}
Answer:"""

    def ask(self, question):
        chunks, chunk_ids = self.retrieve(question)
        prompt = self.build_prompt(chunks, question)

        response = self.llm.chat.completions.create(
            model="gpt-4o-mini",
            messages=[
                {"role": "system", "content": "You are a domain expert assistant. Be concise and accurate. Cite the source when possible."},
                {"role": "user", "content": prompt}
            ],
            temperature=0.0
        )

        answer = response.choices[0].message.content
        input_tok = response.usage.prompt_tokens
        output_tok = response.usage.completion_tokens
        cost = (input_tok * 0.15 / 1_000_000) + (output_tok * 0.60 / 1_000_000)
        self.total_cost += cost

        return {
            "answer": answer,
            "sources": chunk_ids,
            "cost": cost,
            "input_tokens": input_tok,
            "output_tokens": output_tok
        }

    def evaluate(self, test_cases):
        correct = 0
        print("\n--- Evaluation Run ---\n")
        for i, case in enumerate(test_cases):
            result = self.ask(case["question"])
            is_decline = "don't have information" in result["answer"].lower()
            passed = (case["has_answer"] and not is_decline) or (not case["has_answer"] and is_decline)
            if passed:
                correct += 1
            status = "PASS" if passed else "FAIL"
            print(f"  Q{i+1} [{status}]: {case['question'][:60]}...")
            print(f"    A: {result['answer'][:80]}...")
            print(f"    Sources: {result['sources']} | Cost: ${result['cost']:.6f}\n")

        accuracy = correct / len(test_cases) * 100
        print(f"--- Score: {correct}/{len(test_cases)} ({accuracy:.0f}%) ---")
        print(f"--- Total Session Cost: ${self.total_cost:.6f} ---")
        return accuracy

Evaluation & Prompt Iteration

Running your evaluation harness once and calling it done is not enough. The capstone requires you to iterate on your prompts and document the improvement.

Here's what your Prompt Version Log should look like:

Version Change Made Accuracy Notes
v1.0 Basic system prompt: "You are a helpful assistant" 60% Hallucinated on out-of-scope queries. Gave verbose answers.
v1.1 Added grounding instruction: "Answer ONLY from context" 80% Out-of-scope queries now declined. Some answers still too long.
v1.2 Added "Be concise. Max 2 sentences." + few-shot example in system prompt 93% Clean, focused answers. One edge case with ambiguous query still fails.

Why does this matter? Because in a real job, you'll need to prove that your AI system improved. You can't just say "I made it better" — you need the numbers. This log is your evidence.

The minimum requirement is 2 documented iterations showing measurable improvement. More is better.

Demo Review

You will present your capstone to a reviewer (your instructor, a peer, or an SME). Here's what the review covers:

Review Area What You Must Show Pass Criteria
Problem Statement What domain you chose and why. What questions the assistant can answer. Clear, scoped, and realistic.
Architecture Walk through the RAG pipeline: documents → chunks → embeddings → retrieval → generation. Can explain every component and why it's there.
Prompts Show your system prompt and grounding instruction. Explain design decisions. Prompt is well-structured, not a wall of text.
Retrieval Quality Run 3 live queries. Show which chunks were retrieved and why they're relevant. Relevant chunks are consistently in top-3.
Evaluation Report 15-query harness results. Prompt version log with at least 2 iterations. Accuracy ≥ 80%. Documented improvement.
Code Quality Clean, modular code. Reusable functions. Error handling. Reviewer can read and understand the code.

Treat this like a real engineering review. The reviewer will ask follow-up questions: "Why did you choose this chunk size?" or "What happens if the user asks something completely unrelated?" You should be able to answer confidently.
