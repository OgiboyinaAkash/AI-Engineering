Capstone: Production AI Assistant Design Review

1. Capstone Overview

This is the final test. You are tasked with building a production-grade AI assistant that combines everything you've learned in Track 2: robust RAG architecture, agentic tool calling, orchestration, API delivery, and rigorous evaluation.

In Track 1's capstone (C1.6), the focus was on making it work. Here, the focus is on making it scale, defend itself, and prove its quality.

You will build the system, package it, evaluate it using RAGAS, and defend your architectural choices in an Engineering Design Review.

2. Technical Requirements

Your system must meet the following mandatory requirements. Missing any of these will result in an automatic failure in the design review.

Category Requirement
Data & Retrieval - Minimum 100-document corpus (use the expanded dataset provided or your own).
- Persistent Vector DB (ChromaDB or Pinecone).
- Implement at least one metadata filter (e.g., date, category, author).
- Implement a documented chunking strategy (justify your chunk size).
Agentic Behavior - Orchestrate a multi-tool loop (ReAct pattern) or multi-agent workflow.
- Include at least 3 distinct tools (e.g., semantic_search, web_search, calculate).
- Implement hard loop limits (max_iterations) and decision logging.
API & Delivery - Expose the agent via an asynchronous FastAPI endpoint (/ask).
- Implement request logging capturing tokens, cost, latency, and retrieved sources.
- Output must be guarded by at least one structural or content guardrail.
Security - Defend against basic prompt injection using delimiters or input filtering.
- No secrets or API keys hardcoded in the repository.

4. Engineering Design Review

The culmination of this capstone is a live or recorded Engineering Design Review with an SME (Subject Matter Expert). This simulates a real-world tech lead review.

You must be prepared to answer questions such as:

  • Architecture: "Walk me through the lifecycle of a request from the FastAPI endpoint down to the Vector DB and back."
  • Design Choices: "Why did you choose your specific chunk size and overlap? What happened when you tried a smaller chunk?"
  • Evaluation: "What was your RAGAS Faithfulness score? How did you improve it over the baseline?"
  • Safety: "Show me what happens if the LLM enters an infinite loop trying to call a broken tool."
  • Production: "How would this system behave if 1,000 users hit the endpoint simultaneously?"

5. Evaluation Rubric

Criteria Pass Needs Work
System Architecture Clean separation of API, agent, and retrieval layers. Easily readable code. Spaghetti code. Everything in one 800-line file.
Retrieval Quality Metadata filtering is used. Justifiable chunking strategy. Recall@k metrics provided. Blind chunking. No metrics. "It looks okay."
Agent Safety Loop limits enforced. Decision logging implemented. High-risk tools require approval. Agent can loop forever. Errors crash the system.
Evaluation RAGAS evaluation run on ≥20 queries. Results analyzed and documented. No formal evaluation. Tested manually on 3 questions.
Production Readiness Async FastAPI. Cost and latency logged. Dockerized. Secrets managed via .env. Sync endpoints blocking requests. Keys in repo.

Congratulations on reaching the end of the AI Engineering Series! If you pass this design review, you have proven you can build, evaluate, and ship production-grade Applied AI systems.
