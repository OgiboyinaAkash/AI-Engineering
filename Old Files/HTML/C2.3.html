AI Agents, Tool Calling & ReAct Pattern

In C1.4, you taught an LLM to call functions — ask for the weather, calculate a tip, look up a contact. But those were one-shot tool calls: user asks, LLM picks a tool, your code runs it, LLM responds. Done.

Now imagine this: Ravi says to your assistant, "Find the cheapest flight from Bangalore to Mumbai next Friday, check if my calendar is free that day, and if it is, book the flight and send me a confirmation email."

That's not one tool call. That's four — and they depend on each other. You can't book the flight before checking the calendar. You can't send the email before booking. The assistant needs to plan, act, observe the result, and decide what to do next — in a loop.

That's an AI agent.

What Is an AI Agent?

Let's be precise. There's a lot of hype around the word "agent," so let's nail down what it actually means in engineering terms:

An AI agent is a system where an LLM operates in a loop — it reasons about what to do, takes an action (calls a tool), observes the result, and decides the next step. This loop continues until the task is complete or a stopping condition is met.

Feature Simple Tool Calling (C1.4) AI Agent (This Module)
Number of steps Fixed: 1 tool call, 1 response. Variable: loops until task is done.
Decision making LLM picks one tool per query. LLM plans multi-step sequences, adapts based on intermediate results.
State Stateless — each query is independent. Stateful — accumulates context across steps.
Error handling Your code catches errors. The agent itself can retry, try alternative tools, or ask for clarification.
Risk Low — one action, easy to review. Higher — autonomous loops can go wrong.

Think of it like the difference between a vending machine and a personal assistant. A vending machine does one thing when you press a button. A personal assistant takes a goal ("organize my trip"), breaks it into steps, handles surprises along the way, and comes back when it's done.

The ReAct Pattern: Reason + Act

The most widely used agent architecture is called ReAct — short for Reasoning + Acting. The idea is simple: before each action, the LLM explicitly writes out its reasoning. This makes the agent's decision-making transparent and debuggable.

The loop looks like this:

flowchart TD A[Start] --> B[THINK: Reason about state] B --> C{Action needed?} C -- Yes --> D[ACT: Call Tool] D --> E[OBSERVE: Get Result] E --> B C -- No --> F[FINISH: Return Final Answer]

Let's see this concretely. Say the task is: "What's the weather in Bangalore, and is it a good day for cricket?"

Step 1 — THINK: "The user wants weather info for Bangalore and a recommendation.
                  I need to first get the weather data."
Step 1 — ACT:   get_weather(city="Bangalore")
Step 1 — OBSERVE: {"temp": 28, "condition": "Partly Cloudy", "rain_chance": 15}

Step 2 — THINK: "Got the weather. 28°C, partly cloudy, 15% rain chance.
                  That's good cricket weather — low rain, not too hot."
Step 2 — ACT:   FINISH (no more tools needed)
Step 2 — RESPONSE: "Bangalore is 28°C with partly cloudy skies and only
                     15% chance of rain — great day for cricket!"

Notice how the LLM writes out its reasoning at each step? This is what makes ReAct powerful for debugging. When the agent makes a wrong decision, you can read the reasoning and understand why.

Implementing ReAct in Code

import os
import json
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
client = OpenAI()

# --- Tool definitions (same format as C1.4) ---
def get_weather(city):
    data = {"Bangalore": {"temp": 28, "condition": "Partly Cloudy", "rain_chance": 15},
            "Mumbai": {"temp": 33, "condition": "Humid", "rain_chance": 70}}
    return json.dumps(data.get(city, {"temp": 25, "condition": "Unknown", "rain_chance": 50}))

def search_web(query):
    # Mock web search
    results = {"cricket schedule Bangalore": "IPL match at Chinnaswamy Stadium on Saturday at 7 PM.",
               "restaurants near Chinnaswamy": "1. Vidyarthi Bhavan 2. MTR 3. Truffles"}
    return json.dumps({"results": results.get(query, "No results found.")})

tools = [
    {"type": "function", "function": {
        "name": "get_weather",
        "description": "Get current weather for a city.",
        "parameters": {"type": "object", "properties": {"city": {"type": "string"}}, "required": ["city"]}
    }},
    {"type": "function", "function": {
        "name": "search_web",
        "description": "Search the web for information.",
        "parameters": {"type": "object", "properties": {"query": {"type": "string"}}, "required": ["query"]}
    }}
]

available_tools = {"get_weather": get_weather, "search_web": search_web}

# --- The ReAct Agent Loop ---
def run_agent(user_task, max_iterations=5):
    messages = [
        {"role": "system", "content": """You are a helpful assistant that completes tasks step-by-step.
For each step, think about what you need to do next, then use a tool if needed.
When the task is fully complete, respond with your final answer."""},
        {"role": "user", "content": user_task}
    ]

    for iteration in range(max_iterations):
        print(f"\n--- Iteration {iteration + 1} ---")

        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=messages,
            tools=tools,
            temperature=0.0
        )

        message = response.choices[0].message

        # Check if agent wants to call tools
        if message.tool_calls:
            messages.append(message)
            for tool_call in message.tool_calls:
                fn_name = tool_call.function.name
                fn_args = json.loads(tool_call.function.arguments)
                print(f"  ACTION: {fn_name}({json.dumps(fn_args)})")

                result = available_tools[fn_name](**fn_args)
                print(f"  OBSERVE: {result}")

                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": result
                })
        else:
            # No tool call — agent is done
            print(f"  FINAL ANSWER: {message.content}")
            return message.content

    print("  [Agent hit max iterations — stopping.]")
    return "Agent could not complete the task within the iteration limit."

# --- Run it! ---
run_agent("What's the weather in Bangalore and is there a cricket match this weekend?")

Expected output:

--- Iteration 1 ---
  ACTION: get_weather({"city": "Bangalore"})
  OBSERVE: {"temp": 28, "condition": "Partly Cloudy", "rain_chance": 15}

--- Iteration 2 ---
  ACTION: search_web({"query": "cricket schedule Bangalore"})
  OBSERVE: {"results": "IPL match at Chinnaswamy Stadium on Saturday at 7 PM."}

--- Iteration 3 ---
  FINAL ANSWER: The weather in Bangalore is 28°C with partly cloudy skies and
  a low 15% chance of rain. There's an IPL match at Chinnaswamy Stadium this
  Saturday at 7 PM — great weather for it!

The agent made three iterations: two tool calls and one final answer. It figured out on its own that it needed two pieces of information, gathered them sequentially, and synthesized a response. That's the ReAct pattern in action.

Multi-Tool Agents & Planning

Real agents have access to many tools and need to plan which ones to use and in what order. The key insight is that you don't hard-code the plan — the LLM figures it out based on the task and available tools.

Consider a research assistant agent with these tools:

  • search_docs(query) — search internal documents
  • search_web(query) — search the internet
  • summarize(text) — summarize long text
  • save_note(title, content) — save a research note

If Priya asks: "Research our company's Q3 revenue and compare it to the industry average. Save a summary."

The agent might plan:

  1. Search internal docs for "Q3 revenue" → gets the company number
  2. Search the web for "industry average revenue Q3 2026" → gets the benchmark
  3. Summarize both findings into a comparison
  4. Save the summary as a research note

The LLM decided this plan on its own. You just gave it tools and a goal. This is what makes agents powerful — and dangerous, which is why the safety section below is critical.

Error Recovery — When Tools Fail

In production, tools fail. APIs time out. Databases return errors. Search results come back empty. A robust agent doesn't crash when a tool fails — it adapts. 

def run_agent_with_recovery(user_task, max_iterations=7):
    messages = [
        {"role": "system", "content": """You are a helpful assistant. Use tools to complete tasks.
If a tool returns an error or empty result:
1. Try rephrasing the query and calling the tool again.
2. If it fails twice, try an alternative tool.
3. If no tool can help, tell the user what you tried and what failed.
Never make up information to compensate for a failed tool call."""},
        {"role": "user", "content": user_task}
    ]

    for iteration in range(max_iterations):
        response = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=tools, temperature=0.0
        )
        message = response.choices[0].message

        if message.tool_calls:
            messages.append(message)
            for tool_call in message.tool_calls:
                fn_name = tool_call.function.name
                fn_args = json.loads(tool_call.function.arguments)

                try:
                    result = available_tools[fn_name](**fn_args)
                except Exception as e:
                    # Return error to the agent — let IT decide what to do
                    result = json.dumps({"error": str(e), "tool": fn_name, "args": fn_args})
                    print(f"  [Tool Error: {fn_name} failed — {e}]")

                messages.append({"role": "tool", "tool_call_id": tool_call.id, "content": result})
        else:
            return message.content

    return "Agent exceeded maximum iterations."

The key pattern here: don't hide errors from the agent. When a tool fails, return the error message as the tool result. The LLM is surprisingly good at reading error messages and adjusting its approach — rephrasing a search query, trying a different tool, or explaining to the user what went wrong.

What you should never do: silently return fake data when a tool fails. That's how hallucinations get into agent outputs. Always surface the failure.

Agent Safety — This Is Not Optional

Here's the uncomfortable truth: an agent is an LLM running in a loop, autonomously deciding what actions to take. If you don't put guardrails on it, things will go wrong. Not "might." Will.

Real-world agent failures:

  • Infinite loops: Agent keeps calling the same tool with the same arguments because it doesn't understand the result. Your API bill skyrockets.
  • Unauthorized actions: Agent decides to "help" by deleting old records or sending emails to contacts without approval.
  • Data leakage: Agent sends internal company data to an external search API.

Mandatory Safety Measures

Every agent you build in this course must have all four:

Safety Measure Implementation Why It Matters
Loop Limit max_iterations parameter. Hard stop at 5–10. Prevents infinite loops and runaway API costs.
Permission Boundaries Whitelist of allowed tools. Agent cannot access tools outside the list. Prevents unauthorized actions (deleting data, sending emails).
Approval Steps For high-stakes actions (payments, deletions), require human confirmation before execution. Human-in-the-loop for irreversible actions.
Decision Logging Log every reasoning step, tool call, and result to a file. Audit trail. Debug failures. Catch anomalies.

Decision Logging Implementation

import datetime

class AgentLogger:
    def __init__(self, log_file="agent_decisions.jsonl"):
        self.log_file = log_file

    def log(self, iteration, event_type, data):
        entry = {
            "timestamp": datetime.datetime.now().isoformat(),
            "iteration": iteration,
            "type": event_type,  # "think", "action", "observation", "error", "final"
            "data": data
        }
        with open(self.log_file, "a") as f:
            f.write(json.dumps(entry) + "\n")
        print(f"  [{event_type.upper()}] {json.dumps(data)[:100]}...")

# Usage in agent loop:
logger = AgentLogger()

# logger.log(1, "observation", {"result": {"temp": 28, "condition": "Partly Cloudy"}})

Human Approval for High-Stakes Actions

HIGH_RISK_TOOLS = {"send_email", "delete_record", "make_payment"}

def execute_tool(fn_name, fn_args):
    if fn_name in HIGH_RISK_TOOLS:
        print(f"\n⚠️  HIGH-RISK ACTION: {fn_name}({json.dumps(fn_args)})")
        approval = input("  Approve? (yes/no): ").strip().lower()
        if approval != "yes":
            return json.dumps({"error": "Action blocked by user", "tool": fn_name})

    return available_tools[fn_name](**fn_args)

This is the single most important thing in this module. A clever agent with no safety boundaries is a liability, not a feature. Every agent in your capstone must have loop limits, permission boundaries, and decision logging. No exceptions.

4. Starter Code

import os
import json
import random
import datetime
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
client = OpenAI()

# --- Mock Tools ---
def search_docs(query):
    docs_db = {
        "pricing": "CloudNest Starter: ₹499/mo, Pro: ₹1499/mo, Enterprise: ₹4999/mo.",
        "refund": "30-day refund for Pro, 90-day for Enterprise.",
        "support": "24/7 support for Pro and Enterprise plans.",
    }
    for key, val in docs_db.items():
        if key in query.lower():
            return json.dumps({"source": "internal_docs", "result": val})
    return json.dumps({"source": "internal_docs", "result": "No relevant documents found."})

def search_web(query):
    # Simulate 30% failure rate
    if random.random() < 0.3:
        raise ConnectionError("Web search API timed out")
    results = {
        "cloud hosting pricing": "AWS Lightsail: $5-80/mo, DigitalOcean: $4-96/mo, Linode: $5-96/mo",
        "competitor analysis hosting": "Major competitors offer tiered pricing from $5-100/month.",
    }
    for key, val in results.items():
        if any(word in query.lower() for word in key.split()):
            return json.dumps({"source": "web", "result": val})
    return json.dumps({"source": "web", "result": "No results found."})

def summarize(text):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": f"Summarize in 2 sentences:\n{text}"}],
        temperature=0.0
    )
    return json.dumps({"summary": response.choices[0].message.content})

saved_notes = []
def save_note(title, content):
    saved_notes.append({"title": title, "content": content, "saved_at": datetime.datetime.now().isoformat()})
    return json.dumps({"status": "saved", "title": title})

# --- Tool Definitions ---
tools = [
    {"type": "function", "function": {
        "name": "search_docs", "description": "Search internal company documents.",
        "parameters": {"type": "object", "properties": {"query": {"type": "string"}}, "required": ["query"]}
    }},
    {"type": "function", "function": {
        "name": "search_web", "description": "Search the internet for external information.",
        "parameters": {"type": "object", "properties": {"query": {"type": "string"}}, "required": ["query"]}
    }},
    {"type": "function", "function": {
        "name": "summarize", "description": "Summarize a long piece of text into a brief summary.",
        "parameters": {"type": "object", "properties": {"text": {"type": "string"}}, "required": ["text"]}
    }},
    {"type": "function", "function": {
        "name": "save_note", "description": "Save a research note with a title and content.",
        "parameters": {"type": "object", "properties": {
            "title": {"type": "string"}, "content": {"type": "string"}
        }, "required": ["title", "content"]}
    }}
]

available_tools = {
    "search_docs": search_docs, "search_web": search_web,
    "summarize": summarize, "save_note": save_note
}

HIGH_RISK = {"save_note"}

# --- Logger ---
class AgentLogger:
    def __init__(self):
        self.log = []

    def record(self, iteration, event_type, data):
        entry = {"iter": iteration, "type": event_type, "data": data,
                 "time": datetime.datetime.now().isoformat()}
        self.log.append(entry)
        print(f"  [{event_type.upper()}] {str(data)[:120]}")

# --- ReAct Agent ---
def run_research_agent(task, max_iterations=7):
    logger = AgentLogger()
    messages = [
        {"role": "system", "content": """You are a research assistant. Complete the task step by step.
Think about what information you need, use tools to gather it, and synthesize a result.
If a tool fails, try rephrasing or use an alternative. Never fabricate information."""},
        {"role": "user", "content": task}
    ]

    for i in range(max_iterations):
        print(f"\n--- Step {i+1}/{max_iterations} ---")

        response = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=tools, temperature=0.0
        )
        msg = response.choices[0].message

        if msg.tool_calls:
            messages.append(msg)
            for tc in msg.tool_calls:
                fn_name = tc.function.name
                fn_args = json.loads(tc.function.arguments)
                logger.record(i+1, "action", {"tool": fn_name, "args": fn_args})

                # Human approval for high-risk tools
                if fn_name in HIGH_RISK:
                    print(f"\n  ⚠️  Approval needed for: {fn_name}({json.dumps(fn_args)[:80]})")
                    approval = input("  Approve? (yes/no): ").strip().lower()
                    if approval != "yes":
                        result = json.dumps({"error": "Blocked by user"})
                        logger.record(i+1, "blocked", {"tool": fn_name})
                        messages.append({"role": "tool", "tool_call_id": tc.id, "content": result})
                        continue

                try:
                    result = available_tools[fn_name](**fn_args)
                    logger.record(i+1, "observation", {"result": result[:200]})
                except Exception as e:
                    result = json.dumps({"error": str(e)})
                    logger.record(i+1, "error", {"tool": fn_name, "error": str(e)})

                messages.append({"role": "tool", "tool_call_id": tc.id, "content": result})
        else:
            logger.record(i+1, "final", {"answer": msg.content[:200]})
            print(f"\n  FINAL: {msg.content}")
            return msg.content, logger.log

    return "Max iterations reached.", logger.log

# --- Run ---
if __name__ == "__main__":
    answer, log = run_research_agent(
        "Research CloudNest's pricing, find competitor pricing on the web, and save a comparison note."
    )
    print(f"\n--- Agent Decision Log ({len(log)} entries) ---")
    for entry in log:
        print(f"  Step {entry['iter']}: {entry['type']}")
